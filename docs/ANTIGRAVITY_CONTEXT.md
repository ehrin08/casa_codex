# Antigravity Project Context: Sprint Deliverable 1 (SD1)

> Source document: **Sprint Deliverable 1 (SD1) – System Analysis and Design**, Batangas State University, College of Informatics and Computing Sciences.

Use this file as project context for Antigravity when helping create, review, or refine Sprint Deliverable 1 artifacts.

---

## 1. Deliverable Purpose

Sprint Deliverable 1 focuses on analyzing and designing the proposed system through four perspectives:

- **People**
- **Process**
- **Technology**
- **Data**

The deliverable should prioritize professional design artifacts and supporting documentation, not lengthy narrative discussions. The artifacts should establish the foundation for UI/UX design, application development, testing, and deployment in succeeding sprints.

---

## 2. Required SD1 Components

### A. SD0 Revision Compliance

**Print separately using the SD0 template.**

Required evidence:

- Summary of revisions made
- Updated project charter
- Updated risks and challenges assessment

Purpose:

- Demonstrate that all SD0 comments, recommendations, and corrections have been addressed.

---

### B. People Analysis

**Attachment A**

Required artifacts:

- Use Case Diagram
- Use Case Descriptions

Design requirements:

- Identify primary and secondary actors.
- Identify major system functions.
- Use appropriate UML notation.
- Maintain alignment with the project scope and objectives.

---

### C. Process Modeling

**Attachment B**

Required artifact:

- System Flowchart

Design requirements:

- Illustrate major business processes.
- Include decision points where applicable.
- Use standard flowchart notation.
- Maintain logical process flow.

---

### D. Technology Architecture

**Attachment C**

Required artifacts:

- Context Diagram, Level 0
- System Architecture Diagram
- Application Architecture and Technology Stack Diagram
- Deployment Diagram

Design requirements:

- Clearly identify system boundaries.
- Show major software and hardware components.
- Indicate communication pathways.
- Demonstrate deployment environment and infrastructure.

---

### E. Data Modeling

**Attachment D**

Required artifacts:

- Entity Relationship Diagram, ERD
- Entity Definition Table
- Sample Data Tables

Design requirements:

- Identify entities and attributes.
- Indicate primary keys and foreign keys.
- Show relationship cardinalities.
- Ensure consistency with People, Process, and Technology models.
- Include at least **five representative records** for each major transactional and master table.

## 7. Antigravity Working Instructions

When using this project context in Antigravity, prioritize the following:

1. Generate professional system analysis and design artifacts aligned with SD1 requirements.
2. Keep diagrams consistent with one another:
   - Use cases should match processes.
   - Processes should match data flows and technology architecture.
   - ERD entities should support the identified system functions.
3. Avoid long narrative sections unless needed for explanation or defense preparation.
4. Use clear naming conventions for actors, processes, entities, modules, and data tables.
5. Make sure all diagrams and tables are readable and submission-ready.
6. Separate printable artifacts from sections marked **Don't Print**.
7. Prepare defense-ready explanations for every major design decision.
